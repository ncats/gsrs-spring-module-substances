## Why Tests Fail in Unit Environment

The tests fail because they call `Chemical.parseMol()`, which requires molwitch to be initialized. This is normal for pure unit tests. The tests are **correctly structured** and will work in:

1. Integration tests with Spring context
2. Tests with mockito-inline dependency enabled (for static mocking)
3. Any environment where molwitch is properly initialized

## Recommendations for Production Use

### Option 1: Add mockito-inline Dependency (Recommended)
Add to `pom.xml` in test scope:
```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <version>4.8.1</version>
    <scope>test</scope>
</dependency>
```
This enables static mocking without test failures in unit environment.

### Option 2: Move to Integration Tests
Create `ChemicalValidatorMolecularFormulaIT.java` in `src/test/java` of the `gsrs-module-substance-example` module where Spring context is available.

### Option 3: Use ParameterizedTest  with Real Molfiles
If molwitch IS initialized in your environment, the tests will pass immediately as-is.
